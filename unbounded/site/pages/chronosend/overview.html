ChronoSend — Timed Messaging for Every Platform

ChronoSend is a message scheduling application that bridges the gap between intention and delivery. It lets you compose messages once and dispatch them at precisely the right moment — across SMS, email, messaging apps, and webhook endpoints — without writing a single line of integration code.

Why ChronoSend?

Existing scheduling tools are locked inside single platforms. You can schedule an email in Gmail, a tweet in Twitter, a Slack message — but there is no unified tool that treats time as a first-class delivery dimension across every communication channel. ChronoSend fills that gap.

⏰ Time-Aware Delivery

Schedule messages down to the second. Set timezone-aware delivery for recipients anywhere in the world. Recurring schedules, delayed sends, and countdown-triggered releases.

🔗 Cross-Platform

One message, many destinations. Send the same payload to email, SMS, Telegram, Discord, Slack, WhatsApp, and custom webhooks — each formatted for its target channel.

🧩 No-Code Integration

Configure channels through a simple YAML or JSON manifest. No SDKs to install, no API clients to write. Point, schedule, send.

🔐 Privacy-First

Messages are encrypted at rest and in transit. Delivery logs contain metadata only — content is never stored after delivery unless configured for audit trails.

📊 Delivery Analytics

Track open rates, click-through, delivery failures, and timing accuracy per channel. Export logs as CSV or JSON for external analysis.

⚡ Lightweight & Portable

Single binary with no external dependencies. Runs on anything from a Raspberry Pi to a cloud VM. Deploy via Docker, systemd, or as a serverless function.

Supported Channels

📧 Email (SMTP / SendGrid) 💬 SMS (Twilio / Vonage) 📱 Telegram Bot 💎 Discord Webhook 🔷 Slack Webhook 📞 WhatsApp (Business API) 🌐 Custom Webhooks 📄 WebSocket Push

Quick Start

1. Define a channel

# chronosend.yaml
channels:
  alert-email:
    type: smtp
    host: smtp.example.com
    port: 587
    username: alerts@example.com
    password: ${SMTP_PASSWORD}

  dev-team:
    type: slack
    webhook: https://hooks.slack.com/services/xxx

  public-announce:
    type: telegram
    bot_token: ${TELEGRAM_BOT_TOKEN}
    chat_id: "@my_channel"

2. Schedule a message

chronosend send \
  --channels alert-email,dev-team \
  --at "2026-07-15T09:00:00+08:00" \
  --subject "Deployment complete" \
  --body "Version 2.4.1 is live. All systems nominal."

3. Or use the JSON API

curl -X POST https://chronosend.example.com/api/schedule \
  -H "Authorization: Bearer $API_KEY" \
  -d '{
    "channels": ["alert-email", "dev-team"],
    "deliver_at": "2026-07-15T09:00:00+08:00",
    "content": {
      "subject": "Deployment complete",
      "body": "Version 2.4.1 is live. All systems nominal."
    }
  }'

Architecture

ChronoSend uses a simple pipeline architecture: Schedule → Queue → Render → Dispatch → Log.

  1. Schedule — messages are accepted via CLI, REST API, or a recurring cron-like configuration file. Each message is assigned a delivery time and a set of target channels.
  2. Queue — an internal priority queue (backed by SQLite or Redis) holds messages until their delivery time. The scheduler polls the queue every second for due messages.
  3. Render — each channel has a template renderer that transforms the generic message body into the platform-specific format (Markdown for Slack, HTML for email, plain text for SMS, etc.).
  4. Dispatch — rendered messages are handed to channel adapters that handle authentication, rate limiting, retry logic, and delivery confirmation.
  5. Log — every dispatch attempt is recorded with timestamp, channel, status, and response code. Failed deliveries are retried with exponential backoff.

The entire pipeline is designed to be observable. Each stage emits structured logs that can be consumed by Grafana, Loki, or any log aggregator.

Use Cases

  • Incident response — page the on-call engineer via SMS, Slack, and email simultaneously when a monitoring alert fires.
  • Marketing campaigns — schedule drip sequences across email and SMS with precise timing for different time zones.
  • Appointment reminders — send confirmations and reminders via the patient's preferred channel 24 hours and 1 hour before an appointment.
  • DevOps automation — notify multiple teams across different platforms when a deployment, backup, or maintenance window completes.
  • Personal productivity — schedule a message to your future self, a birthday greeting, or a recurring daily standup prompt.

Status

ChronoSend is currently in design phase. The architecture is specified, the channel adapter interface is defined, and a proof-of-concept scheduler has been prototyped. Public alpha is targeted for Q3 2026.

If you want to contribute — whether by writing a channel adapter, improving the scheduler, or documenting the API — see the contribution guide or reach out through the portal.
